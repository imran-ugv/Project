What is HTML?

HTML (HyperText Markup Language) was created by Tim Berners-Lee in 1991 as a standard for creating web pages. It's a markup language used to structure content on the web, defining elements like headings, paragraphs, links, and images. HTML forms the backbone of web content. In layman's terms, HTML is like the skeleton of a website. It's a set of instructions that tells a web browser how to display text, images, videos, and other elements on a webpage. Think of it as the building blocks that create the structure and look of a website, similar to how bricks and mortar are used to build a house.

In a nutshell:

  • HTML is the language of the web, used to create websites.
  • HTML defines the barebone structure or layout of web pages that we see on the Internet.
  • HTML consists of a set of tags contained within an HTML document, and the associated files typically have either a .html or .htm extension.
  • There are several versions of HTML, with HTML5 being the most recent version.

Features of HTML

  • It is platform-independent. For example, Chrome displays the same pages identically across different operating systems such as Mac, Linux, and Windows.
  • Images, videos, and audio can be added to a web page (For example - YouTube shows videos on their website).
  • HTML is a markup language and not a programming language.
  • It can be integrated with other languages like CSS, JavaScript, etc. to show interactive (or dynamic) web pages.

Why the Term HyperText & Markup Language?

The term 'Hypertext Markup Language' is composed of two main words: 'hypertext' and 'markup language.' 'Hypertext' refers to the linking of text with other documents, while 'markup language' denotes a language that utilizes a specific set of tags.

Thus, HTML is the practice of displaying text, graphics, audio, video, etc., in a certain way using special tags.

Note: Tags are meaningful texts enclosed in angle braces, like '<...>'. For example, the '' tag. Each tag has a unique meaning and significance in building an HTML page, and it can influence the web page in various ways.

CSS adds the paint and finishing touches, making the car look appealing with color, style, and design. JavaScript is similar to the engine and mechanical parts, infusing the car with functionality, movement, and interactive features. Similarly, when developing a website, HTML lays out the structure, CSS enhances its visual appeal, and JavaScript provides interactivity and dynamic content.

History of HTML

  • In 1989, Tim Berners-Lee established the World Wide Web (www), and in 1991, he created the first version of HTML.
  • From 1995 to 1997, further work was done to develop and refine different versions of HTML.
  • In 1999, a committee was organized that standardized HTML 4.0, a version still used by many today.
  • The latest and most stable version of HTML is 5, also known as HTML5.

HTML Working

You must have heard of frontend and backend. Frontend refers to the visible part of a website or app that users interact with, like the tables, images, and buttons. It's built using languages like HTML, CSS, and JavaScript. The backend, on the other hand, handles behind-the-scenes operations like storing and processing data when users interact with the frontend. It uses languages like Python, Ruby, and Java. In essence, frontend is what users see, while backend manages the functionality.

How does HTML work?

When we want to access any information on the internet, we search for it using a web browser. The web browser retrieves the content from web servers, where it is stored in the form of HTML documents.

An HTML document is created by writing code with specific tags in a code editor of your choice. The document is then saved with the '.html' extension. Once saved, the browser interprets the HTML document, reads it, and renders the web page.

The text editor has the HTML code of a website. This website can now be viewed in a beautifully rendered format using a computer program known as a web browser.

What is a Web Browser?

A web browser is a program that understands HTML tags and renders them in a human-readable format that is easily viewable by people visiting the website. Developers write code in HTML because it's a straightforward way to instruct the web browser on what to display. In the next section, I'll show you how to set up VS Code for writing your own HTML code and rendering it in a web browser.

What is an HTML Document?

An HTML document is a text document saved with the '.html' or '.htm' extension, containing text and specific tags enclosed in '< >'. These tags provide the necessary instructions for configuring the web page. The tags themselves are standardized and fixed. The structure of an HTML document will be explained later in this HTML tutorial.

What is a Rendered Page:

The rendered page is the output screen of our HTML Document which is the page displayed on the browser.

How does a basic website work?

  • Web Browser(client) requests websites like www.codewithharry.com from the web server.
  • Web server in return sends HTML, CSS, and JavaScript files to it.
  • HTML, CSS, and JavaScript files are parsed by a web browser which is responsible for showing you a beautiful website.

How does a browser work?

A web browser plays a crucial role in parsing and rendering HTML code to the client. A web browser is a complex program that performs many tasks behind the scenes. Here's a summary of how it works:

  • A browser is an application that reads HTML documents and displays them as web pages. Browsers can't access the content directly from where it's stored; this is where servers come into play.
  • A server acts as an intermediary, listening to browser requests and fulfilling them by delivering the HTML document to the browser.
  • Web browsers perform two main tasks: parsing and rendering.
  • During the parsing stage, the browser receives raw bytes, which are converted into characters. These characters are then converted into tokens, which in turn are transformed into nodes. These nodes are organized into a tree-like data structure known as the DOM (Document Object Model).
  • Once the DOM tree is constructed, the browser moves on to the rendering stage. At this point, each node in the DOM tree is rendered and displayed on the screen.

HTML Page Structure

HTML (Hypertext Markup Language) provides the basic structure for web pages. An HTML document has a well-defined structure that outlines how a page is organized. Below is a breakdown of the HTML page structure:

1. Doctype Declaration

The <!DOCTYPE html> declaration tells the browser that the document is an HTML5 document. It should always be the first thing in your HTML file.

<!DOCTYPE html>

2. HTML Tag

The <html> tag is the root element of the HTML document. All other elements are contained within it.

Example:

<html lang="en">

lang="en" specifies that the language of the document is English.

3. Head Section

The <head> section contains meta-information about the HTML document, such as its title, character encoding, and linked external resources like CSS stylesheets or scripts.

Meta Tags: Provide information about the document (such as character set and author).

Title Tag: Defines the title of the document, which appears in the browser tab.

Link Tag: Used to link external resources such as stylesheets.

Script Tag: Used to link or embed JavaScript files (although typically placed just before the closing </body> tag).

Example of the Head Section:

<head> <meta charset="UTF-8"> <meta name="description" content="An example of HTML page structure"> <meta name="author" content="Your Name"> <title>HTML Page Structure</title> <link rel="stylesheet" href="styles.css"> </head>

4. Body Section

The <body> section contains the content that will be visible to the users on the web page. This includes text, images, links, and other elements that make up the main content of the page.

Example of the Body Section:

<body> <header> <h1>Welcome to HTML Page Structure</h1> </header> <section> <p>This is an example of how to structure an HTML page.</p> </section> <footer> <p>© 2025 Your Name. All rights reserved.</p> </footer> </body>

5. Footer Section

The <footer> tag typically appears at the bottom of the page and contains information such as copyright, privacy policy links, or contact information.

Example of a Footer Section:

<footer> <p>© 2025 Your Name. All rights reserved.</p> </footer>

Complete Example:

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <meta name="description" content="An example of HTML page structure"> <meta name="author" content="Your Name"> <title>HTML Page Structure</title> <link rel="stylesheet" href="styles.css"> </head> <body> <header> <h1>Welcome to HTML Page Structure</h1> </header> <section> <p>This is an example of how to structure an HTML page.</p> </section> <footer> <p>© 2025 Your Name. All rights reserved.</p> </footer> </body> </html>

Key Points to Remember:

  • The <!DOCTYPE html> declaration is required to ensure the document is interpreted as HTML5.
  • The <html> element is the root element of the HTML page.
  • The <head> section contains metadata about the document, such as the title and links to external resources like stylesheets.
  • The <body> section contains the actual content displayed on the webpage.
  • The <header>, <footer>, and <section> tags help organize content semantically.

HTML Tags

HTML tags are the building blocks of an HTML document. Tags are used to define elements on the page. Every HTML document is made up of different tags that indicate how the content should be displayed.

🔹 What Are HTML Tags?

An HTML tag is used to mark the beginning and the end of an HTML element. Tags usually come in pairs, with an opening tag and a closing tag, although some tags do not have a closing tag.

Example of a Tag:

<p>This is a paragraph.</p>

🔹 Types of HTML Tags:

  • Empty Tags (Self-closing tags): These tags do not require a closing tag. They are used to insert elements like images, line breaks, etc.
  • Paired Tags: These tags come in pairs: an opening tag and a closing tag. The closing tag is the same as the opening tag but with a forward slash (/) before the tag name.

🔹 Common HTML Tags:

  • <html>: The root element that contains all the content of the web page.
  • <head>: Contains meta-information about the document like title, character encoding, etc.
  • <body>: Contains the visible content of the page, like text, images, etc.
  • <h1> - <h6>: Heading tags, used to define headings. <h1> is the largest, and <h6> is the smallest.
  • <p>: Paragraph tag, used to define blocks of text.
  • <b>: Bold text.
  • <i>: Italic text.
  • <u>: Underlined text.
  • <a>: Anchor tag, used to create hyperlinks.
  • <img>: Image tag, used to embed images.
  • <ul>: Unordered list.
  • <ol>: Ordered list.
  • <li>: List item, used inside <ul> or <ol>.

🔹 Example of HTML Tags in Action:

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>HTML Tags Example</title> </head> <body> <h1>Understanding HTML Tags</h1> <p>This is a <b>bold</b> paragraph with <i>italic</i> and <u>underlined</u> text.</p> <a href="https://www.example.com">Visit Example Website</a> <img src="image.jpg" alt="An example image"> </body> </html>

🧠 Pro Tips:

  • Always make sure to use paired tags where needed.
  • For tags like <img>, use the alt attribute to improve accessibility.
  • HTML tags are case-insensitive, but it's a good practice to use lowercase letters.
  • Avoid leaving tags open or unclosed, as this can lead to layout issues.

HTML Elements

HTML elements are the building blocks of a web page. They define the structure and content of the page. An HTML element typically consists of an opening tag, content, and a closing tag. In some cases, an element might be self-closing, meaning it doesn't require a closing tag.

🔹 What Are HTML Elements?

An HTML element is a container for content and is defined by tags. Each element can have attributes that provide additional information about it, and these attributes are placed inside the opening tag.

Structure of an HTML Element:

<tagname>Content goes here</tagname>

🔹 Types of HTML Elements:

  • Structural Elements: These elements help organize the layout and structure of the webpage.
  • Text Formatting Elements: These elements are used to format text and control its appearance.
  • Link and Media Elements: These elements link to other resources or display media like images, audio, and video.
  • List Elements: These elements are used to create different types of lists.
  • Form Elements: These elements are used for creating forms.

🔹 Example of HTML Elements in Action:

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>HTML Elements Example</title> </head> <body> <h1>Understanding HTML Elements</h1> <p>This is an example of using HTML elements to structure content.</p> <a href="https://www.example.com">Visit Example Website</a> <img src="image.jpg" alt="Example Image"> <ul> <li>Item 1</li> <li>Item 2</li> <li>Item 3</li> </ul> <form> <input type="text" placeholder="Enter your name"> <button type="submit">Submit</button> </form> </body> </html>

🧠 Pro Tips:

  • Use semantic HTML elements for better accessibility and SEO.
  • Always close your tags properly, especially for non-self-closing elements.
  • Use the correct tags for the content type, e.g., <a> for links and <img> for images.
  • Remember that some elements are self-closing, such as <img> and <br>, and do not need closing tags.

HTML Attributes

HTML attributes provide additional information about an element. They are always placed inside the opening tag and usually come in name-value pairs. Attributes help define the behavior or appearance of elements on the webpage.

🔹 What Are HTML Attributes?

HTML attributes define specific properties for an element. They are used to configure or modify the behavior of HTML elements. Common attributes include id, class, src, href, and style.

Syntax of HTML Attributes:

<tagname attribute="value">Content goes here</tagname>

🔹 Common HTML Attributes:

  • id Attribute: The id attribute assigns a unique identifier to an element. This can be used to apply specific styles or JavaScript functionality.
  • class Attribute: The class attribute is used to assign one or more class names to an element. This allows you to apply CSS styles to multiple elements.
  • href Attribute: The href attribute is used to specify the URL of a link.
  • src Attribute: The src attribute specifies the source file for an image, audio, or video.
  • alt Attribute: The alt attribute provides an alternative text description for an image, which is important for accessibility.
  • style Attribute: The style attribute is used to apply inline CSS to an element.

🔹 Example of HTML Attributes in Action:

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>HTML Attributes Example</title> </head> <body> <h1 id="main-header">Welcome to HTML Attributes</h1> <p class="highlight">This paragraph has a class attribute applied.</p> <a href="https://www.example.com" target="_blank">Visit Example</a> <img src="image.jpg" alt="An example image" width="300"> <p style="font-size: 20px;">This paragraph has inline CSS applied.</p> </body> </html>

🧠 Pro Tips:

  • Always use meaningful values for attributes like id and class for better readability and maintainability.
  • For better accessibility, use the alt attribute in all image tags.
  • The href attribute in anchor tags is essential for navigation, and target="_blank" opens links in a new tab.
  • Use the style attribute sparingly and prefer external stylesheets for maintainability.

HTML Comments

HTML comments are used to insert notes or explanations in your HTML code. Comments are not displayed in the browser and are purely for the developers' reference. They are useful for explaining sections of the code, leaving reminders, or temporarily disabling certain code without deleting it.

🔹 Syntax of HTML Comments:

<!-- This is a comment -->

Comments can span multiple lines:

<!-- This is a multi-line comment. It can span across multiple lines. -->

🔹 Why Use HTML Comments?

  • Code Documentation: Comments can help explain what the code is doing, making it easier for others (or yourself) to understand the logic when revisiting the code later.
  • Temporarily Disable Code: You can comment out code to temporarily disable it during testing or debugging without deleting it.
  • Provide Instructions or Reminders: Comments are often used to leave instructions or reminders for other developers who may be working on the code.

🔹 Example of HTML Comments:

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>HTML Comments Example</title> </head> <body> <h1>Welcome to HTML Comments</h1> <!-- This is a single-line comment --> <p>This paragraph is visible in the browser.</p> <!-- This is a multi-line comment. It can be used to provide additional information. The following section of the page is under development. --> <p>More content coming soon!</p> </body> </html>

🧠 Pro Tips:

  • Use comments to explain complex code or to clarify your intent.
  • Avoid over-commenting simple or self-explanatory code.
  • Be cautious not to leave sensitive or unnecessary information in comments.
  • HTML comments are visible in the source code but not rendered in the browser.

HTML Id & Classes

In HTML, the id and class attributes are used to identify elements and apply styles or JavaScript functionality to them. They are essential for styling, accessibility, and manipulating elements with JavaScript.

🔹 HTML id Attribute

The id attribute uniquely identifies an HTML element. It must be unique within a document, meaning no two elements can have the same id.

Usage: The id is commonly used to target a specific element for styling or to reference it in JavaScript.

Example of id in HTML:

<div id="header">This is a unique header</div>

You can style an element with a specific id using CSS:

#header { background-color: lightblue; padding: 10px; font-size: 20px; }

🔹 HTML class Attribute

The class attribute is used to define a group of elements that share the same styles or functionality. Unlike id, a class can be reused across multiple elements.

Example of class in HTML:

<p class="intro">This is an introductory paragraph.</p> <p class="intro">This is another introductory paragraph.</p>

You can style all elements with the same class using CSS:

.intro { color: darkblue; font-family: Arial, sans-serif; }

🔹 Combining id and class

You can use both id and class together on the same element. This allows you to take advantage of both unique and reusable styles or functionality.

Example of both id and class:

<div id="main-header" class="header-style">This is a main header</div>

🔹 Key Differences Between id and class:

Feature id class
Uniqueness Must be unique within a page Can be reused on multiple elements
Purpose Identifies a single element Identifies a group of elements
Targeting in CSS #id selector .class selector
JavaScript Usage document.getElementById("id") document.getElementsByClassName("class")

🧠 Pro Tips:

  • Use id when you want to uniquely identify an element (e.g., for anchor links or JavaScript manipulation).
  • Use class when you need to apply the same style or behavior to multiple elements.
  • Both id and class can be used together, but the id should remain unique on the page.

Skeletal Tags

Skeletal tags refer to the fundamental building blocks that form the basic structure or "skeleton" of an HTML document. These tags are always present in every HTML file and help define the document layout, content, and metadata.

📋 Common Skeletal Tags:

Tag Description
<!DOCTYPE html> Defines the document as HTML5
<html> Root element of the HTML document
<head> Contains meta-information about the page
<title> Specifies the title shown in the browser tab
<meta> Provides metadata such as charset or description
<link> Links external resources like stylesheets
<script> Includes or links JavaScript files
<body> Contains the visible content of the page

💡 Example Structure:

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>>Document Skeleton</title> <link rel="stylesheet" href="styles.css"> </head> <body> <h1>Welcome to Skeletal Tags</h1> <p>This is the foundation of an HTML document.</p> <script src="script.js"></script> </body> </html>

🧠 Key Points:

  • Every HTML document must begin with <!DOCTYPE html>.
  • The <html> element wraps all content on the page.
  • The <head> section includes information that doesn't appear visually.
  • The <body> tag holds all visible content.
  • Use <meta>, <link>, and <script> for structured layout and enhanced functionality.

Heading Tags

In HTML, heading tags are used to define headings in a webpage. They help structure the content hierarchically and improve both readability and SEO (Search Engine Optimization).

🔸 Types of Heading Tags:

Tag Description
<h1> Defines the main heading (most important)
<h2> Sub-heading under <h1>
<h3> Sub-heading under <h2>
<h4> Sub-heading under <h3>
<h5> Sub-heading under <h4>
<h6> Least important heading

🎯 Key Points:

  • Only <h1> tag should be used once per page for better SEO.
  • Heading tags are block-level elements and take up the full width.
  • They are styled larger and bolder by default.
  • Headings help search engines and screen readers understand structure.

🧠 Pro Tips:

  • Use headings to create a logical outline of your page.
  • Do not skip heading levels (e.g., don’t go from <h1> to <h4>).
  • Use with semantic HTML for better accessibility.

🧪 Example of Heading Tags:

<h1>This is a Main Heading</h1> <h2>This is a Sub-heading</h2> <h3>This is a Smaller Sub-heading</h3> <h4>This is a Level 4 Heading</h4> <h5>This is a Level 5 Heading</h5> <h6>This is the Least Important Heading</h6>

HTML Paragraphs Tag

The HTML <p> tag defines a paragraph. Browsers automatically add some space (margin) before and after each <p> element to separate them from surrounding content.

🔹 Basic Syntax:

<p>This is a paragraph in HTML.</p>

✅ Key Features of <p> Tag:

Feature Description
Tag Name <p>
Paired/Unpaired Paired (<p>...</p>)
Display Block-level element
Default Style Browsers add vertical spacing before and after

💡 Example:

<p>HTML is the standard language for creating web pages.</p>

🧪 Real-life Example:

<p>I went to Cox’s Bazar last winter. The sea beach was beautiful, and the weather was perfect.</p>
<p>We enjoyed the sunset and delicious seafood at night. It was an unforgettable experience!</p>

⚠️ Notes:

  • Never nest block-level elements (like <div>) inside <p> tags.
  • Use multiple <p> tags to create clearly separated text blocks.
  • You can style <p> tags using CSS to change font, size, color, etc.

🧠 Pro Tips:

  • Use paragraphs to improve readability of your content.
  • Combine with CSS for better control over line spacing and alignment.

HTML Horizontal Line Tag

The <hr> tag is used to insert a horizontal line (or rule) in a web page. It is commonly used to divide content into sections, acting as a visual separator. This tag is self-closing and doesn’t require a closing tag.

🔹 Key Characteristics:

  • Represents a thematic break or content division.
  • It is a block-level and self-closing element.
  • Can be styled using CSS (width, color, height, etc.).

📌 Syntax:

<hr>

🧪 Example:

<h2>Welcome to My Blog</h2>
<p>This is my first blog post. I hope you find it helpful!</p>

<hr>

<h2>Another Post</h2>
<p>Here’s another article with useful tips on HTML and CSS.</p>

🧠 Real-Life Use:

Think of the <hr> tag as a visual divider — like a line separating two chapters in a book. It helps guide the reader by making sections clear and neat.

HTML Line Break Tag

The <br> tag in HTML is used to insert a line break in the text, meaning the content after it will appear on the next line. It's a self-closing tag and doesn't require a closing tag like most other HTML elements.

✅ Why Use <br>?

HTML normally ignores line breaks in the code. The <br> tag forces the browser to start a new line, which is useful in addresses, poems, or anywhere structured text is required.

📋 Key Features

Feature Description
Tag Name <br>
Self-closing Yes
Purpose Inserts a line break
Display Type Inline

📋 Key Features of <a> Tag

Feature Description
Tag <a>
Attribute href (defines the destination URL)
Closing Tag Yes, it must be closed with </a>
Default Style Blue color and underlined text
Use Cases Linking to web pages, files, email, phone numbers

Introduction to HTML Forms

Forms are an essential part of web development because they allow users to interact with a website by submitting data. Whether it's logging in, registering for an account, submitting feedback, or making a payment, forms help collect and send user input to a web server for processing.

HTML forms consist of various elements like text fields, radio buttons, checkboxes, drop-down lists, and buttons. These elements help users provide their information in a structured way.

Why Are Forms Important?

Forms are critical for interactive web applications and provide various functionalities such as:

  • User Registration: Collecting user details for creating accounts.
  • Feedback Submission: Enabling users to provide feedback on services.
  • Search Features: Allowing users to search for content within a website.
  • Payment Processing: Collecting payment details for transactions.

HTML Form Structure

An HTML form is created using the <form> element, which contains different input fields like text boxes, radio buttons, checkboxes, and more. Here's the basic structure of an HTML form:

<form action="submit-form.php" method="POST"> <label for="name">Name:</label> <input type="text" id="name" name="name" required> <label for="email">Email:</label> <input type="email" id="email" name="email" required> <label for="message">Message:</label> <textarea id="message" name="message" rows="4" cols="50" required></textarea> <input type="submit" value="Submit"> </form>

Example: Contact Form

Here’s an example of a simple contact form where users can submit their name, email, and message:

<form action="submit-form.php" method="POST"> <h2>Contact Us</h2> <label for="name">Full Name:</label> <input type="text" id="name" name="name" required> <label for="email">Email Address:</label> <input type="email" id="email" name="email" required> <label for="message">Message:</label> <textarea id="message" name="message" rows="5" required></textarea> <input type="submit" value="Submit"> </form>

Key Form Elements

1. Input Fields

The <input> element is used to collect text data from users, like a username, password, etc.

<input type="text" id="username" name="username" placeholder="Enter your username" required> <input type="password" id="password" name="password" placeholder="Enter your password" required>

Expected Output for Input Fields:

The user will be able to fill in a username and password. The password field will hide the characters for privacy.

2. Radio Buttons

Radio buttons allow the user to select one option from a set.

<label for="male">Male</label> <input type="radio" id="male" name="gender" value="male"> <label for="female">Female</label> <input type="radio" id="female" name="gender" value="female">

Expected Output for Radio Buttons:

Gender selection: The user can only choose one option, either "Male" or "Female".

3. Checkboxes

Checkboxes allow users to select one or more options.

<label for="subscribe">Subscribe to Newsletter</label> <input type="checkbox" id="subscribe" name="subscribe">

Expected Output for Checkboxes:

The user can choose to subscribe to the newsletter by checking the box.

4. Drop-down Menu

A drop-down menu allows users to select one option from a list of choices.

<label for="country">Country:</label> <select id="country" name="country"> <option value="usa">United States</option> <option value="canada">Canada</option> <option value="uk">United Kingdom</option> </select>

Expected Output for Drop-down Menu:

The user can select their country from a drop-down list of options.

Form Submission

Once the user fills out the form, they can submit it by clicking the Submit button. The data will be sent to the URL specified in the action attribute using the HTTP method (GET or POST) defined in the method attribute.
